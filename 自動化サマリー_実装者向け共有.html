🪄 自動化している主要な機能・フロー

① 応募検知と自動書類判定プロセスの実行

媒体からの応募データを管理シート（DB）に自動転記します。同時に「自動書類判定システム」が稼働し、判定結果をシートに自動反映します。
※ 面接・説明会通過の場合は即座に次の日程調整フローへ移行します。AIやルールエンジンが判定を持ち越した「保留」ステータスになった場合のみ、人間のオペレーターに判断を委ねるように組んでいます。

自動転記 自動合否判定

② 面接官カレンダーの可用性チェッカー＆自動マッチング

書類選考を通過した候補者に対して、面接官（または説明会枠）の Google カレンダーを API 経由で探索します。空きスロットが見つかった場合は即時にカレンダーイベントと Google Meet の URL を自動生成し、データベースに紐付けます。もし空き枠が見つからない場合は、自動的に次のフロー（候補者へリスケフォーム送信、またはLINE通知）へアサインされます。

Google Calendar API Meet URL生成 マッチングエンジン

③ ステータス駆動型のメール配信エンジン

「書類選考不合格」「面接確定」「リマインド」などのステータス遷移をフックとして、ステータスに応じた適切なテンプレートでメール（Gmail）を自動送信します。
なお、面接や説明会の前日（朝10時）には一括でリマインダーを送信するバッチ処理が動いており、逆に深夜帯（22時〜翌8時）は候補者への新規案内メールをキューイングしてブロックし、翌朝に送信する制御を入れています。

Gmail App API 夜間送付ブロッキング バッチ配信

④ AIによる面接ログの自動生成と評価連携

面接官が Google Meet の録画と文字起こし機能をONにして面接を実施すると、面接終了後に生成された文字起こしテキストを API 経由でサーバー側で取得します。この文字起こしデータを Gemini モデルへプロンプトとともに渡し、面接の所感・評価、候補者の特徴などの要点を抽出した「申し送りログ」を自動作成します。
その後、生成されたログテキストは即時に社内の連携用 LINE グループへ自動通知（共有）されます。

⑤ バックグラウンドでの例外監視・アラート機構

基本はフルオートですが、「面接官候補が全員埋まっている」「AIログのパースに失敗した」といった例外処理（Exception）や、人間の介入が必要なフェーズ（書類判定保留ステータス等）が発生した際は、即座に LINE Messaging API 経由で専用チャネルにアラートを投げる監視・通知機構を備えています。これにより運用側はアラートが鳴った時だけ作業すれば済むようになります。
